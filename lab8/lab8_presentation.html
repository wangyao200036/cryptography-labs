Цель работы

Изучить теоретические основы арифметических операций с целыми числами многократной точности, реализовать алгоритмы сложения, вычитания, умножения и деления больших целых чисел, провести тестирование и анализ их эффективности.

Ход лабораторной работы

Основные понятия

• Большие целые числа: числа, превышающие стандартные типы данных компьютера
• b-ичная система счисления: представление числа в виде u=u1u2...unu = u_1u_2...u_n по основанию b≥2b \ge 2
• Многократная точность: арифметические операции с числами произвольной длины
• Алгоритмы: детерминированные процедуры для выполнения арифметических операций
• Криптографическое значение: операции с большими числами лежат в основе современных криптосистем

Алгоритм сложения

Вход: u=u1u2...unu = u_1u_2...u_n, v=v1v2...vnv = v_1v_2...v_n, основание bb
Выход: w=w0w1...wnw = w_0w_1...w_n (с переносом)

Псевдокод: 1. j:=nj := n, k:=0k := 0 (перенос) 2. wj:=(uj+vj+k)modbw_j := (u_j + v_j + k) \mod b 3. k:=⌊(uj+vj+k)/b⌋k := \lfloor (u_j + v_j + k) / b \rfloor 4. j:=j−1j := j - 1, если j>0j > 0 → шаг 2 5. w0:=kw_0 := k

Пример: 999+1=1000999 + 1 = 1000 (десятичная система)
Сложность: O(n)O(n)

Алгоритм вычитания

Вход: u=u1u2...unu = u_1u_2...u_n, v=v1v2...vnv = v_1v_2...v_n, основание bb
Выход: w=w1w2...wn=u−vw = w_1w_2...w_n = u - v

Псевдокод: 1. j:=nj := n, k:=0k := 0 (kk — заём) 2. wj:=(uj−vj+k)modbw_j := (u_j - v_j + k) \mod b 3. k:=⌊(uj−vj+k)/b⌋k := \lfloor (u_j - v_j + k) / b \rfloor 4. j:=j−1j := j - 1, если j>0j > 0 → шаг 2 5. w0:=kw_0 := k

Пример: 1000−1=9991000 - 1 = 999 (десятичная система)
Сложность: O(n)O(n)
Требование: u≥vu \ge v

Алгоритм умножения столбиком

Вход: u=u1u2...unu = u_1u_2...u_n, v=v1v2...vmv = v_1v_2...v_m, основание bb
Выход: w=w1w2...wm+nw = w_1w_2...w_{m+n}

Принцип работы: - Классический метод “школьного” умножения - Каждая цифра uiu_i умножается на каждую цифру vjv_j - Результаты суммируются с учётом позиций

Пример: 123×456=56088123 \times 456 = 56088
Сложность: O(n⋅m)O(n \cdot m)

Алгоритм быстрого умножения

Вход: u=u1u2...unu = u_1u_2...u_n, v=v1v2...vmv = v_1v_2...v_m, основание bb
Выход: w=w1w2...wm+nw = w_1w_2...w_{m+n}

Оптимизация: - Суммирование произведений для каждого s=i+js = i + j - Более эффективная организация вычислений - Снижение количества промежуточных операций

Формула: wm+n−s=∑i+j=sun−i⋅vm−s+imodb w_{m+n-s} = \sum_{i+j=s} u_{n-i} \cdot v_{m-s+i} \mod b

Сложность: O(n⋅m)O(n \cdot m) (лучшая константа)

Алгоритм деления

Вход: u=un...u1u0u = u_n...u_1u_0, v=vt...v1v0v = v_t...v_1v_0, n≥t≥1n \ge t \ge 1, vt≠0v_t \neq 0
Выход: Частное q=qn−t...q0q = q_{n-t}...q_0, остаток r=rt...r0r = r_t...r_0

Ключевые шаги: 1. Нормализация чисел 2. Поразрядное деление с коррекцией частного 3. Проверка и исправление отрицательных остатков 4. Формирование окончательного результата

Пример: 12345÷67=18412345 ÷ 67 = 184 (остаток 17)
Сложность: O(n2)O(n^2)

Криптографические применения

• RSA: умножение и возведение в степень больших чисел
• Эллиптические кривые: операции с точками требуют арифметики больших чисел
• Diffie-Hellman: модульные операции с большими простыми числами
• DSA/ECDSA: генерация и проверка подписей
• Оптимизация: эффективные алгоритмы критичны для производительности криптосистем

Ограничения и оптимизации

Ограничения: - Линейные алгоритмы неэффективны для очень больших чисел - Память растёт пропорционально длине чисел - Деление остаётся самой сложной операцией

Оптимизации: - Использование быстрого преобразования Фурье (FFT) для умножения - Блочные алгоритмы для работы с памятью - Параллельные вычисления для ускорения операций - Аппаратная поддержка (специальные процессоры)
